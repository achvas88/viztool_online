jQuery SexySelect v. 0.5

Author : Nico VanHaaster
Updated: 20/12/2010

The jQuery SexySelect is a simple plugin written for educational purposes to allow for easier (and frankly prettier) manipulation of Single or Multiple select boxes (drop down lists).

This plugin will only work with select boxes and no other element types.

Features

  • Skinned using jQuery UI themes 1.8.6+
  • Collapsable select region
  • Allow user to add & remove options
  • Single, Multiple & No Selection modes
  • Auto Sorting
  • Drag & Drop Sorting (with Auto Sorting disabled)
  • Events On Item Creation & Items Syncronization
  • Custom Header Text
  • Simple item count & validation methods
  • Watermark Entry Text
  • Custom CSS Styles such as height, width, and background display.

Change Log

Changes in v0.5
  • Reworked Selectors for optimal performance
  • Added Filtering the items
  • Added drag and drop across lists with one or two way binding
  • Added class tags to all elements for simpler skinning
  • Removed code duplications and added new binding methods
  • Created a drag handler to mimimize the element while inflight.
  • Included 3 new plugins. (see plugins : requires v0.5 & stylize pack)
    • Checkbox: Restyled Checkbox, replaces the default look
    • Radio Box: creates a radio button (list) with a Checkbox look and Tri State radio buttons
    • .disableTextSelect - Disable Text Select Plugin : Copyright (c) 2007 James Dempster (letssurf@gmail.com, http://www.jdempster.com/category/jquery/disabletextselect/
  • Added Options: (see options for more details)
    • styleize : allow for style the radio & checkboxes
    • confirmDelete: creates a dialog to confirm removal with custom text, option jQuery dialog and callBack handler
    • connectList : connect your list with another for cross list drag and drop with optional two-way binding
    • filterKeyUpTimeout : timeout for filtering (searching)
    • triStateRadio : create tristate radio buttons
    • allowFilter : allowFiltering
  • Created Methods : (see methods for more details)
    • Filter : filter the results against filtered text, optional programically supply the filter text
    • addItem : Add a new item to the original list
    • syncAux : Auxiliary call to sync the items
    • connectWith : Allow connections to be made programically
    • totalVisibleItems : Returns a count of all the items visible (not hidden by the filter)
    • removeItem : Removes a specific item by value
Changes in v0.4
  • Added method : elements. This returns all the option elements that are currently added to the select list
  • Added method : selectOption. This method accepts two values, the element to be modified and the selected state. (true/false)
Changes in v0.3
  • Renamed option : _debugShowOriginal to allowDebug. This option also will show alert messages on errors through custom events.
  • Added option : nooptionstext which allows for you to change the text when no options are added to the list, default is 'no options added'
  • Added event : onItemSelect : runs a custom event when an option is selected, returns the element created and a collection of all the options in the list. Runs after the click event is handled
  • Added event : onitemcreating : runs a custom event before the item is created. This allows for custom validation. This event passes in the input text object and also the EventArgs, this event expects the EventArgs to be passed back. See the Events Tab for more info
  • Added method : showError : Shows the error bar with custom text that you pass in. Expects the text to display and the timeout to slide up. See the methods tab
